  <action>
1. Run `go get fyne.io/fyne/v2@v2.7.3` to add the Fyne dependency.

2. Create `internal/gui/assets/cluckers_logo.png` -- a simple placeholder logo image (a solid-color circle or chicken silhouette, 256x256 PNG). This will be replaced with the real logo later. Can be generated programmatically or use a minimal PNG.

3. Create `internal/gui/assets/embed.go` with `//go:build gui` tag:
   - Package `assets`
   - `//go:embed cluckers_logo.png` into `var LogoData []byte`
   - Export `LogoResource()` function returning `fyne.NewStaticResource("cluckers_logo.png", LogoData)`

4. Create `internal/gui/detect.go` (NO build tag -- shared types only if needed, or empty placeholder).

5. Create `internal/gui/detect_linux.go` with `//go:build gui && linux`:
   - `func CanShowGUI() bool` -- returns true if `os.Getenv("DISPLAY") != ""` OR `os.Getenv("WAYLAND_DISPLAY") != ""`

6. Create `internal/gui/detect_windows.go` with `//go:build gui && windows`:
   - `func CanShowGUI() bool` -- always returns true (Windows always has GUI capability)

7. Create `internal/gui/theme.go` with `//go:build gui` tag:
   - Define `cluckersTheme` struct implementing `fyne.Theme` interface
   - Dark color scheme: background ~#1E1E23, button ~#3C3C46, primary accent green ~#4CAF50 (Material green)
   - Delegate to `theme.DefaultTheme()` with `theme.VariantDark` for any unset values
   - Export `NewCluckersTheme()` constructor

8. Create `internal/gui/app.go` with `//go:build gui` tag:
   - `func Run(cfg *config.Config) error` -- main entry point for GUI mode
   - Create Fyne app via `app.New()`, set custom theme via `a.Settings().SetTheme(NewCluckersTheme())`
   - Create window "Cluckers" via `a.NewWindow("Cluckers")`
   - For now, set placeholder content: the embedded logo centered with a "Cluckers Central" label below it (this will be replaced by login/main screens in plan 02/03)
   - Handle Steam Deck detection: if `isSteamDeck()` (reuse existing detection logic from `internal/launch/deckconfig.go` or check for Steam Deck env markers), call `w.SetFullScreen(true)`. Otherwise `w.Resize(fyne.NewSize(480, 640))`.
   - Call `w.ShowAndRun()` and return nil
   - Note: isSteamDeck detection on Linux can check `/sys/devices/virtual/dmi/id/board_vendor` for "Valve" or check `SteamDeck` env var. On Windows, always return false.

9. Create `internal/cli/root_gui.go` with `//go:build gui` tag:
   - In `init()`, set `rootCmd.RunE` to a function that:
     a. Checks `gui.CanShowGUI()` -- if false, return `cmd.Help()`
     b. If true, calls `gui.Run(Cfg)` and returns the result
   - Import `github.com/0xc0re/cluckers/internal/gui`

10. Create `internal/cli/root_nogui.go` with `//go:build !gui` tag:
    - Empty `init()` or no init at all -- Cobra's default behavior shows help when no subcommand is given, which is the existing behavior

IMPORTANT: All files in `internal/gui/` (except detect.go if it has no build tag) MUST have `//go:build gui` to keep the CLI-only build path working with CGO_ENABLED=0. The `internal/gui/assets/` package also needs the gui build tag.

IMPORTANT: Fyne does NOT require CGO on Windows. On Linux it does. The build command for GUI is: `CGO_ENABLED=1 go build -tags gui -o cluckers ./cmd/cluckers` (Linux) or `go build -tags gui -o cluckers.exe ./cmd/cluckers` (Windows).
  </action>
