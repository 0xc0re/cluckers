Use `launch.StepPending`, `launch.StepRunning`, `launch.StepDone`, `launch.StepFailed`, `launch.StepSkipped` directly. This avoids type duplication between launch and widgets packages.
   - Define `StepItem` struct: `Name string`, `Status launch.StepStatus`
   - Define `StepListWidget` struct:
     - Embeds `widget.BaseWidget`
     - `items []StepItem` (internal state)
     - `container *fyne.Container` (the actual rendered VBox)
   - `NewStepListWidget(stepNames []string) *StepListWidget`:
     - Create items array with all steps set to `launch.StepPending` status
     - Build initial container as `container.NewVBox()` with a row per step
     - Each row: `container.NewHBox(icon, label)` where icon is a pending indicator (e.g., `widget.NewIcon(theme.MediaRecordIcon())` or a circle) and label is `widget.NewLabel(stepName)`
   - `func (s *StepListWidget) UpdateStep(name string, status launch.StepStatus)`:
     - Find step by name, update its status
     - Rebuild the icon for that row:
       - Pending: grey circle icon (`theme.MediaRecordIcon()`)
       - Running: play/spinner icon (`theme.MediaPlayIcon()`) -- or a highlighted label
       - Done: green checkmark (`theme.ConfirmIcon()`)
       - Failed: red X (`theme.ErrorIcon()`)
       - Skipped: grey checkmark
     - Call `s.container.Refresh()` to update display
   - `func (s *StepListWidget) GetContainer() *fyne.Container` -- returns the internal container for embedding in layouts
   - Note: Since Fyne does not support animated spinners natively in widgets, use icon changes + label color changes to indicate "running" state (e.g., bold text or highlighted color for the active step)

2. Create `internal/launch/reporter_gui.go` with `//go:build gui` tag:
   - Package `launch`
   - Define `GUIReporter` struct with field `updateFn func(name string, status StepStatus)`
   - Implement `ProgressReporter` interface:
     - `StepStarted(name)`: calls `fyne.Do(func() { g.updateFn(name, StepRunning) })`
     - `StepCompleted(name)`: calls `fyne.Do(func() { g.updateFn(name, StepDone) })`
     - `StepFailed(name, err)`: calls `fyne.Do(func() { g.updateFn(name, StepFailed) })`
     - `StepSkipped(name)`: calls `fyne.Do(func() { g.updateFn(name, StepSkipped) })`
     - `StepPaused(name)`: no-op in GUI (progress bar not needed -- download progress shown via step status)
   - Constructor: `NewGUIReporter(updateFn func(name string, status StepStatus)) *GUIReporter`

IMPORTANT: All `updateFn` calls MUST be wrapped in `fyne.Do()` because the pipeline runs in a background goroutine.

IMPORTANT: The `StepStatus` type is defined ONCE in `internal/launch/reporter.go` (from plan 04-02). The widgets package imports it from there. The GUIReporter in `internal/launch/` uses it directly since it's in the same package.
