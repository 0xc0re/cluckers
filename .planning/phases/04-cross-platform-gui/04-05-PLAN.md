   - Keep existing CGO_ENABLED=0 builds for CLI-only variant (Linux + Windows)
   - Add Fyne build dependencies step (before GUI builds):
     ```yaml
     - name: Install Fyne build dependencies
       run: |
         sudo apt-get update
         sudo apt-get install -y gcc libgl1-mesa-dev xorg-dev libxkbcommon-dev
     ```
   - Add GUI build steps:
     ```yaml
     - name: Build GUI (Linux)
       env:
         CGO_ENABLED: 1
       run: go build -tags gui -o /dev/null ./cmd/cluckers

     - name: Build GUI (Windows cross-compile)
       env:
         CGO_ENABLED: 0
         GOOS: windows
       run: go build -tags gui -o /dev/null ./cmd/cluckers
     ```
   - Note: Windows GUI cross-compile uses CGO_ENABLED=0 because Fyne does NOT need CGO on Windows. The `gui` build tag just includes the GUI Go files; no C compiler needed for Windows.
   - Add GUI vet steps:
     ```yaml
     - name: Vet GUI (Linux)
       env:
         CGO_ENABLED: 1
       run: go vet -tags gui ./...

     - name: Vet GUI (Windows)
       env:
         CGO_ENABLED: 0
         GOOS: windows
       run: go vet -tags gui ./...
     ```
   - Keep existing test step (`go test ./...`) -- tests don't require GUI tag
   - Reorder steps logically: checkout, setup go, install mingw, build shm_launcher, install fyne deps, build CLI (linux), build CLI (windows), build GUI (linux), build GUI (windows), test, vet CLI, vet GUI

IMPORTANT: The `Install Fyne build dependencies` step must come before any CGO_ENABLED=1 build. The mingw install step is already there for shm_launcher.

<task type="auto">
  <name>Task 2: Update goreleaser and release workflow for GUI binaries</name>
  <files>
    .goreleaser.yaml
    .github/workflows/release.yaml
  </files>
  <action>
1. Update `.goreleaser.yaml`:
   - Keep existing `cluckers` build target (CLI-only, CGO_ENABLED=0, linux+windows)
   - Add a second build target for GUI:
     ```yaml
     builds:
       - id: cluckers-cli
         main: ./cmd/cluckers
         binary: cluckers
         env:
           - CGO_ENABLED=0
         goos:
           - linux
           - windows
         goarch:
           - amd64
         ldflags:
           - -s -w -X main.version={{.Version}} -X main.commit={{.ShortCommit}} -X main.date={{.Date}}

       - id: cluckers-gui-linux
         main: ./cmd/cluckers
         binary: cluckers-gui
         env:
           - CGO_ENABLED=1
         goos:
           - linux
         goarch:
           - amd64
         tags:
           - gui
         ldflags:
           - -s -w -X main.version={{.Version}} -X main.commit={{.ShortCommit}} -X main.date={{.Date}}

       - id: cluckers-gui-windows
         main: ./cmd/cluckers
         binary: cluckers-gui
         env:
           - CGO_ENABLED=0
         goos:
           - windows
         goarch:
           - amd64
         tags:
           - gui
         ldflags:
           - -s -w -X main.version={{.Version}} -X main.commit={{.ShortCommit}} -X main.date={{.Date}}
     ```
   - Update archives section to produce separate archives:
     ```yaml
     archives:
       - id: cli
         builds:
           - cluckers-cli
         format: tar.gz
         name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"
         format_overrides:
           - goos: windows
             format: zip

       - id: gui
         builds:
           - cluckers-gui-linux
           - cluckers-gui-windows
         format: tar.gz
         name_template: "{{ .ProjectName }}-gui_{{ .Version }}_{{ .Os }}_{{ .Arch }}"
         format_overrides:
           - goos: windows
             format: zip
     ```
   - Keep checksum and changelog sections as-is

2. Update `.github/workflows/release.yaml`:
   - Add Fyne build dependencies step (same as CI):
     ```yaml
     - name: Install Fyne build dependencies
       run: |
         sudo apt-get update
         sudo apt-get install -y gcc libgl1-mesa-dev xorg-dev libxkbcommon-dev gcc-mingw-w64-x86-64
     ```
     Note: Combine the mingw install with Fyne deps into one step for efficiency
   - The goreleaser action should handle the rest -- goreleaser will use the build tags and CGO settings from `.goreleaser.yaml`
   - Ensure the goreleaser action has access to the C compiler by NOT setting CGO_ENABLED=0 as a top-level env (remove the `env: CGO_ENABLED: 0` from job level, let each build target control its own)

IMPORTANT: goreleaser v2 supports per-build `env` settings. The CGO_ENABLED is set per build target, not globally.

IMPORTANT: Windows GUI build uses CGO_ENABLED=0 because Fyne does not need CGO on Windows. The `gui` build tag includes the GUI Go source files which compile to pure Go on Windows.

NOTE: If goreleaser cannot handle mixed CGO_ENABLED builds in a single run (some targets need CGO=1, others CGO=0), an alternative approach is to use `goreleaser-cross` Docker image which has cross-compilation toolchains. But try the simpler approach first -- goreleaser should handle per-build env vars.
  </action>
