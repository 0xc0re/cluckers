  <name>Proton-GE Detection and Compatdata Health</name>
  <files>internal/wine/proton.go, internal/wine/proton_test.go, internal/wine/compatdata.go, internal/wine/compatdata_test.go</files>
  <behavior>
    TDD Feature 1: FindProton(configOverride string) (*ProtonGEInstall, error)

    Detection priority (per locked user decision):
    1. Bundled: CLUCKERS_BUNDLED_PROTON env var -> validate proton script exists at $CLUCKERS_BUNDLED_PROTON/proton
    2. Config override: configOverride param -> resolve to ProtonDir (if path points to wine64, go up 3 dirs; if path points to proton script or dir, use directly) -> validate proton script exists
    3. System scan: call existing FindProtonGE(home) -> return first result if any
    4. Not found: return *ui.UserError with per-distro install instructions via ProtonInstallInstructions(distro)

    Cases:
    - CLUCKERS_BUNDLED_PROTON="/app/proton/GE-Proton10-1" with valid proton script -> returns ProtonGEInstall{ProtonDir: "/app/proton/GE-Proton10-1", WinePath: "/app/proton/GE-Proton10-1/files/bin/wine64"}
    - CLUCKERS_BUNDLED_PROTON set but proton script missing -> warn, fall through to config/system
    - configOverride="/path/to/GE-Proton10-1/files/bin/wine64" -> returns ProtonGEInstall{ProtonDir: "/path/to/GE-Proton10-1", WinePath: configOverride}
    - configOverride="/path/to/GE-Proton10-1" (directory with proton script) -> returns ProtonGEInstall{ProtonDir: configOverride, WinePath: configOverride + "/files/bin/wine64"}
    - No bundled, no config, system scan finds GE-Proton10-1 -> returns that install
    - Nothing found on arch -> error.Message contains "Proton-GE not found", Suggestion contains "pacman" and "ProtonUp-Qt"
    - Nothing found on ubuntu -> Suggestion contains "apt" or "ProtonUp-Qt"

    Version warning (per locked decision: "Old Proton-GE versions: Warn but allow"):
    - GE-Proton7-55 detected -> FindProton succeeds, caller can check version via DisplayVersion() and parse for warning

    TDD Feature 2: ProtonGEInstall methods

    - ProtonScript() -> filepath.Join(ProtonDir, "proton")
    - DisplayVersion() -> filepath.Base(ProtonDir) e.g. "GE-Proton10-1"

    TDD Feature 3: CompatdataHealthy(compatdataPath string) bool

    Cases:
    - compatdataPath exists and pfx/drive_c is a directory -> true
    - compatdataPath does not exist -> false
    - compatdataPath exists but pfx/ missing -> false
    - compatdataPath exists, pfx/ exists, but drive_c missing -> false

    TDD Feature 4: CompatdataPath() string

    - Returns filepath.Join(config.DataDir(), "compatdata")

    TDD Feature 5: ProtonInstallInstructions(distro string) string

    Per locked decision: per-distro Proton-GE install instructions (no Wine fallback)
    - "arch" / "steamos" -> mentions ProtonUp-Qt, yay/paru for AUR
    - "ubuntu" / "debian" -> mentions ProtonUp-Qt
    - "fedora" -> mentions ProtonUp-Qt
    - default -> links to GE GitHub
  </behavior>
