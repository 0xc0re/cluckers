  <name>Proton Environment and Command Construction</name>
  <files>internal/launch/proton_env.go, internal/launch/proton_env_test.go</files>
  <behavior>
    TDD Feature 1: buildProtonEnv(compatDataPath string, verbose bool) []string

    Takes the current os.Environ() and produces a filtered+augmented env slice for proton run.

    Cases:
    - Input env contains LD_LIBRARY_PATH="/appimage/lib" -> output does NOT contain LD_LIBRARY_PATH
    - Input env contains WINEPREFIX="/old/prefix" -> output does NOT contain WINEPREFIX
    - Input env contains WINE="/usr/bin/wine" -> output does NOT contain WINE
    - Input env contains WINEDLLOVERRIDES="something" -> replaced with "dxgi=n"
    - Input env contains WINEFSYNC="1" -> output does NOT contain WINEFSYNC
    - Input env contains WINEESYNC="1" -> output does NOT contain WINEESYNC
    - Output contains STEAM_COMPAT_DATA_PATH={compatDataPath}
    - Output contains SteamGameId=0
    - Output contains SteamAppId=0
    - Output contains STEAM_COMPAT_CLIENT_INSTALL_PATH= (empty value)
    - Output contains WINEDLLOVERRIDES=dxgi=n
    - verbose=true -> output contains PROTON_LOG=1
    - verbose=false -> output does NOT contain PROTON_LOG
    - Unrelated env vars (HOME, PATH, USER) pass through unchanged

    TDD Feature 2: filterEnv(env []string, keys ...string) []string

    Removes env vars matching any of the given key names.

    Cases:
    - filterEnv(["HOME=/home/user", "LD_LIBRARY_PATH=/lib", "PATH=/bin"], "LD_LIBRARY_PATH") -> ["HOME=/home/user", "PATH=/bin"]
    - filterEnv(["A=1", "B=2", "C=3"], "A", "C") -> ["B=2"]
    - filterEnv(["A=1"], "X") -> ["A=1"] (no match, unchanged)

    TDD Feature 3: buildProtonCommand(protonScript string, shmPath string, bootstrapPath string, shmName string, gameExe string, gameArgs []string) (program string, args []string)

    Constructs the python3 proton run command with correct argument ordering.

    Cases -- with SHM (bootstrap present):
    - protonScript="/opt/GE-Proton10-1/proton", shmPath="/tmp/shm_launcher.exe", bootstrapPath="/tmp/bootstrap.bin", shmName="Local\\realm_content_bootstrap_1234", gameExe="/home/user/.cluckers/game/Realm-Royale/Binaries/Win64/ShippingPC-RealmGameNoEditor.exe", gameArgs=["-user=foo", "-token=bar", "-hostx=1.2.3.4"]
    - Returns program="python3", args=["/opt/GE-Proton10-1/proton", "run", "/tmp/shm_launcher.exe", "Z:\\tmp\\bootstrap.bin", "Local\\realm_content_bootstrap_1234", "Z:\\home\\user\\.cluckers\\game\\Realm-Royale\\Binaries\\Win64\\ShippingPC-RealmGameNoEditor.exe", "-user=foo", "-token=bar", "-hostx=1.2.3.4"]
    - Note: shmPath is a Linux path (proton converts), bootstrapPath and gameExe are Wine Z: paths (consumed by Windows processes)

    Cases -- without SHM (no bootstrap):
    - shmPath="", bootstrapPath="", shmName="" -> program="python3", args=[protonScript, "run", gameExe, gameArgs...]
    - Note: gameExe here is still a Linux path since there's no shm_launcher intermediary

    TDD Feature 4: protonErrorSuggestion(compatDataPath string) string

    Returns actionable error suggestion text per locked user decision.

    Cases:
    - compatDataPath="/home/user/.cluckers/compatdata" -> returns string containing:
      "1. Delete /home/user/.cluckers/compatdata/ and relaunch"
      "2. Update Proton-GE to latest version"
      "3. Run `cluckers update` to verify game files"

    TDD Feature 5: shmBridgeError(exitErr error, stderr string, compatDataPath string) *ui.UserError

    Detects shm_launcher-specific failures and returns a distinct UserError separate from general Proton crashes (per locked decision: "SHM bridge failures: Distinct error message separate from general Proton failures").

    Detection approach: Inspect stderr for shm_launcher patterns (e.g., "CreateFileMapping", "OpenFileMapping", "shm_launcher", "shared memory") OR check if the exit code indicates shm_launcher crashed before the game started (exit code non-zero and stderr contains shm_launcher patterns).

    Cases:
    - stderr contains "CreateFileMapping failed" -> returns UserError with Message "Shared memory bridge failed", Suggestion mentioning "try deleting {compatDataPath} and relaunching"
    - stderr contains "shm_launcher" and "error" -> returns UserError with Message "Shared memory bridge failed"
    - stderr contains "OpenFileMapping" -> returns UserError with Message "Shared memory bridge failed"
    - stderr does NOT contain any shm_launcher patterns -> returns nil (caller falls through to general protonErrorSuggestion)
    - exitErr is nil -> returns nil (no failure to detect)

    TDD Feature 6: lastNLines(s string, n int) string

    Returns the last n lines of a string.

    Cases:
    - lastNLines("a\nb\nc\nd\ne", 3) -> "c\nd\ne"
    - lastNLines("ab", 5) -> "ab" (fewer lines than n)
    - lastNLines("", 3) -> ""

  <implementation>
    Create internal/launch/proton_env.go (Linux-only, //go:build linux) containing:

    1. **filterEnv(env []string, keys ...string) []string**: Iterate env, split on "=" to get key, skip if key matches any of the provided keys. Return filtered slice.

    2. **buildProtonEnv(compatDataPath string, verbose bool) []string**: Call filterEnv on os.Environ() to strip LD_LIBRARY_PATH, WINEPREFIX, WINE, WINEDLLOVERRIDES, WINEFSYNC, WINEESYNC. Append required Proton vars: STEAM_COMPAT_DATA_PATH, STEAM_COMPAT_CLIENT_INSTALL_PATH (empty), SteamGameId=0, SteamAppId=0, WINEDLLOVERRIDES=dxgi=n. Conditionally append PROTON_LOG=1 if verbose.

    NOTE: For testability, also create a variant buildProtonEnvFrom(baseEnv []string, compatDataPath string, verbose bool) []string that accepts the base env as a parameter. buildProtonEnv calls this with os.Environ(). Tests use buildProtonEnvFrom directly for deterministic input.

    3. **buildProtonCommand(protonScript, shmPath, bootstrapPath, shmName, gameExe string, gameArgs []string) (string, []string)**: If shmPath is non-empty (bootstrap mode): return "python3", [protonScript, "run", shmPath, wine.LinuxToWinePath(bootstrapPath), shmName, wine.LinuxToWinePath(gameExe), gameArgs...]. If shmPath is empty (no bootstrap): return "python3", [protonScript, "run", gameExe, gameArgs...]. The first positional arg to proton run (the exe to launch) uses a Linux path -- proton converts it internally. But paths that are game argument values (bootstrap file, game exe in shm_launcher args) must be Wine Z: paths because they are consumed by Windows processes running under Wine.

    4. **protonErrorSuggestion(compatDataPath string) string**: Return formatted multi-line suggestion string with the 3 common fixes from locked decisions.

    5. **shmBridgeError(exitErr error, stderr string, compatDataPath string) *ui.UserError**: If exitErr is nil, return nil. Search stderr (case-insensitive) for shm_launcher patterns: "CreateFileMapping", "OpenFileMapping", "shm_launcher", "shared memory". If any match, return &ui.UserError{Message: "Shared memory bridge failed", Detail: lastNLines(stderr, 10), Suggestion: "Try deleting " + compatDataPath + " and relaunching. If the problem persists, run 'cluckers update' to verify game files."}. If no match, return nil (caller handles as general Proton error).

    6. **lastNLines(s string, n int) string**: Split on newline, take last n elements, rejoin.

    Test file uses standard Go testing. For buildProtonEnvFrom tests, construct deterministic base env slices. For buildProtonCommand tests, verify exact argument ordering and path conversion.
  </implementation>
</feature>
