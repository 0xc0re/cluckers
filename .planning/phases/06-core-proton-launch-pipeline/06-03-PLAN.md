  <name>Task 2: Rewrite Linux pipeline steps and LaunchGame for Proton</name>
  <files>internal/launch/pipeline_linux.go, internal/launch/process_linux.go</files>
  <action>
    **pipeline_linux.go -- Replace Wine steps with Proton steps:**

    Replace the 3 Wine steps (Detecting Wine, Ensuring Wine prefix, Verifying Wine prefix) with 2 Proton steps:

    ```go
    func platformSteps(_ *LaunchState) []Step {
        return []Step{
            {Name: "Detecting Proton", Fn: stepDetectProton},
            {Name: "Preparing Proton environment", Fn: stepEnsureCompatdata},
        }
    }
    ```

    Keep platformPostSteps unchanged (Deck config still needed).

    **stepDetectProton implementation:**
    - Call `wine.FindProton(state.Config.WinePath)` (config WinePath serves as override for Proton too)
    - On success: set `state.ProtonScript = install.ProtonScript()`, `state.ProtonDir = install.ProtonDir`, `state.ProtonDisplayVersion = install.DisplayVersion()`
    - Also set `state.WinePath = install.WinePath` for backward compat (process_windows.go pattern)
    - Show version: `ui.Verbose(fmt.Sprintf("Proton: %s (%s)", install.DisplayVersion(), install.ProtonDir), state.Config.Verbose)`
    - Version warning for old Proton-GE (per locked decision): parse version from directory name using `wine.ProtonVersionRe`, warn if major < 9: `ui.Warn(fmt.Sprintf("%s detected, version 9+ recommended", install.DisplayVersion()))`
    - Note: `protonVersionRe` is unexported in detect.go. Either export it or duplicate the check. Better to use the exported ProtonGEInstall.Version() method and compare. Or simply re-parse with a local regex. Simplest: export the regex by renaming to ProtonVersionRe. But that changes detect.go which is not in our files_modified... Instead, just use a local regex match in the pipeline step. The regex pattern is simple: `GE-Proton(\d+)-(\d+)`.

    **stepEnsureCompatdata implementation:**
    - Get compatdata path: `compatdata := wine.CompatdataPath()`
    - Call `wine.CompatdataHealthy(compatdata)`
    - If healthy: set `state.CompatDataPath = compatdata`, verbose log, return nil
    - If NOT healthy:
      - If directory exists (damaged): `ui.Warn("Proton environment damaged, recreating...")`, `os.RemoveAll(compatdata)` (per locked decision)
      - Create directory: `os.MkdirAll(compatdata, 0755)`
      - Show first-launch message (per locked decision): the spinner step name is already "Preparing Proton environment" -- for first launch, change the step name dynamically via reporter or just log a message: `ui.Info("Preparing Proton environment (first launch only)...")`
      - Set `state.CompatDataPath = compatdata`
    - Note: The actual prefix creation happens inside `proton run` automatically when STEAM_COMPAT_DATA_PATH points to an empty/new directory. We just need to ensure the directory exists.

    **process_linux.go -- Rewrite LaunchGame for Proton:**

    Replace the entire LaunchGame function with Proton invocation:

    1. Validate game exe exists (same as current)
    2. Build game args (same as current -- these are consumed by the Windows game process)
    3. Extract shm_launcher and write bootstrap (same pattern as current)
    4. Build proton command using `buildProtonCommand()` from proton_env.go (Plan 02)
    5. Build proton env using `buildProtonEnv()` from proton_env.go (Plan 02)
    6. Set up stderr capture: `var stderrBuf bytes.Buffer`. If verbose, use `io.MultiWriter(os.Stderr, &stderrBuf)`. If not verbose, use `&stderrBuf` only.
    7. Stdout: if verbose, os.Stdout. If not verbose, nil (suppress Proton noise).
    8. On failure: first check for SHM-specific failure via `shmBridgeError(err, stderrBuf.String(), cfg.CompatDataPath)`. If it returns non-nil, return that distinct UserError. Otherwise, fall through to general error: extract last 10 lines via `lastNLines(stderrBuf.String(), 10)`, return UserError with Message "Proton launch failed", Detail with last 10 stderr lines, and Suggestion from protonErrorSuggestion(cfg.CompatDataPath). This implements the locked decision: "SHM bridge failures: Distinct error message separate from general Proton failures."
    9. On Ctrl+C (ctx.Err != nil): return nil (same as current).

    Key differences from current code:
    - Command: `exec.CommandContext(ctx, "python3", protonArgs...)` instead of `exec.CommandContext(ctx, wine64, ...)`
    - Env: `buildProtonEnv(cfg.CompatDataPath, cfg.Verbose)` instead of manual WINEPREFIX/WINEFSYNC/WINEDLLOVERRIDES
    - No more LD_LIBRARY_PATH filtering here (buildProtonEnv handles it)
    - No more wine log file at /tmp/cluckers_wine.log (replaced by stderr buffer)
    - No more wine.IsProtonGE check for WINEFSYNC (Proton handles this internally)

    Remove old imports that are no longer needed (if any). Add new imports: "bytes".
