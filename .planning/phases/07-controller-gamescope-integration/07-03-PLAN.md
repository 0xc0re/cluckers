<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Hardware validation of controller persistence on Steam Deck</name>
  <what-built>
    Plans 01-02 implemented Steam installation detection and wired SteamGameId + STEAM_COMPAT_CLIENT_INSTALL_PATH into the Proton launch environment. The launcher now:
    1. Detects the Steam installation directory (FindSteamInstall)
    2. Reads shortcuts.vdf to find the cluckers non-Steam shortcut app ID
    3. Sets SteamGameId and SteamAppId to the resolved app ID (instead of 0)
    4. Sets STEAM_COMPAT_CLIENT_INSTALL_PATH to the detected Steam root

    This should cause Proton Wine to set X11 window class hints as `steam_app_{appid}`, allowing Gamescope to track the game window across D3D recreation during UE3 ServerTravel.
  </what-built>
  <how-to-verify>
    Prerequisites:
    - Build cluckers from current branch: `go build -o cluckers ./cmd/cluckers`
    - Transfer binary to Steam Deck
    - Ensure cluckers is added as a non-Steam game in Steam (run `cluckers steam add` if needed)

    Test steps:
    1. SSH to Steam Deck: `ssh deck@192.168.20.103`
    2. Run with verbose to confirm env vars: `./cluckers launch -v`
    3. Verify in verbose output:
       - "Steam: /home/deck/.local/share/Steam" (or similar)
       - "Steam shortcut app ID: XXXXXXX" (a non-zero numeric value)
    4. In-game: navigate to lobby with controller (A/B/X/Y buttons, bumpers, triggers should work)
    5. Queue into a match
    6. After the loading screen (ServerTravel transition), verify:
       - A/B/X/Y buttons still respond
       - Bumpers (LB/RB) still work
       - Triggers (LT/RT) still fire
       - Joystick axes still control movement/camera

    Expected result: Controller buttons persist through the transition.

    If controller drops (fallback investigation):
    - Check X11 window properties: `xprop -root | grep STEAM_GAME` during lobby and after transition
    - Check Gamescope focus: `xprop -root | grep GAMESCOPE_FOCUSED_APP` before and after
    - Check window class: `xprop WM_CLASS` on the game window -- should show `steam_app_{appid}`
    - If WM_CLASS shows `steam_app_0`, the SteamAppId env var is not being read. Check if SteamGameId vs SteamAppId naming matters.
    - Log Proton output: `PROTON_LOG=1 ./cluckers launch -v 2>&1 | tee proton.log`
  </how-to-verify>
  <resume-signal>
    Type "approved" if controller persists through ServerTravel, or describe the failure:
    - "buttons drop" -- which buttons, when exactly (loading screen, match start, etc.)
    - "env vars wrong" -- what verbose output showed
    - "game didn't launch" -- error message
  </resume-signal>
