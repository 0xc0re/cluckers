<context>
The proxy architecture: reads evdev events from Steam Input virtual pad (event8), applies dead reckoning for button hold during ServerTravel, writes to uinput virtual Xbox 360 pad (event11). EVIOCGRAB should prevent Wine/SDL from seeing the Steam Input pads directly.

The most likely root cause is that `evdev.Open(sourceFile)` doesn't work correctly with a pre-opened file descriptor, OR the EVIOCGRAB ioctl signature is wrong (uintptr vs uint cast), OR the run() goroutine isn't starting/is panicking.

Key code path: `proxy.go:Start()` → opens source fd → EVIOCGRAB → `evdev.Open(sourceFile)` → creates uinput → `go p.run(ctx)`. The run() goroutine calls `p.sourceDev.Poll()` in a loop.

The `kenshaw/evdev` library's `Open()` function wraps an existing `*os.File` — it should NOT re-open the file. But verify this.
</context>
