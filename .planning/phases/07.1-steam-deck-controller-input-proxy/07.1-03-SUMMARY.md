## Accomplishments
- Complete proxy Run loop: reads evdev events via kenshaw/evdev Poll, applies dead reckoning for button hold during ServerTravel, inverts Y axes, forwards to uinput sink
- Two new pipeline steps: stepPatchWinebus (idempotent Wine registry patching) and stepStartInputProxy (Steam Deck only, non-fatal)
- InputProxyCleanup wired through LaunchState -> LaunchConfig -> LaunchGame defer chain
- Proxy lifecycle: NewInputProxy -> Start (detect + create virtual device + goroutine) -> Stop (signal + wait + destroy + close)
- Windows build fully unaffected: InputProxyCleanup is nil, no Linux-only imports in cross-platform files

## Decisions Made
- Used kenshaw/evdev Poll() for event reading: provides context-aware channel with automatic cleanup on context cancellation, typed event envelopes, handles SYN_DROPPED
- Proxy is non-fatal on all systems: if Start() fails, the pipeline step logs a warning and continues. Game works without proxy but may have controller drops during ServerTravel on Steam Deck
- IsSteamDeck gate prevents proxy from starting on desktop Linux where it is unnecessary
- Added 100ms sleep after virtual device creation to allow udev to register the device before the game opens it
- Proxy cleanup is prepended to the cleanup chain (runs before temp file removal) for prompt goroutine shutdown
