ROOT CAUSE (CONFIRMED — not cluckers, not this machine)
-------------------------------------------------------
- 6744 aborts at startup with, in Binaries/Logs/MCTS-*.log:
      Bad marshal [UNKNOWN] id=[0/0x00000000] flags=0x00 size=594600 place=61   Source=Platform
  then system-*.log: "Abnormal termination triggered by abort call".
  It dies BEFORE UE3 engine init, so no RealmGame/Logs/Launch.log is written.
- Only TWO files changed 6742 -> 6744:
      Binaries/Win64/ShippingPC-RealmGameNoEditor.exe
      Binaries/Win64/MctsInterface.dll   <- the DLL that emits "Bad marshal"
  Every other MCTS/EOS/EAC DLL (CSMctsInterface, CSMctsBridge, solstice,
  AgentInterface, EOSSDK-Win64-Shipping, eac_server64) is byte-identical.
- The 594600-byte object is RealmGame/Localization/WordFilter.dat — the only
  file of that size — and it is BYTE-IDENTICAL to 6742's. So 6744's new
  MctsInterface.dll rejects data that 6742's DLL parsed fine.
- The cluckers install is BYTE-PERFECT to manifest-v0.37.6744.0.json
  (0 files missing, 0 extra; only RealmSystemSettings.ini differs by +4 bytes,
  which the game rewrites at runtime). Nothing for a launcher to fix.
- PROOF: dropping 6742's MctsInterface.dll into the 6744 install clears the
  "Bad marshal" and the game reaches a window — then the 6744 .exe fatal-errors
  (matched exe/DLL pair; you cannot mix versions). See UE3AutoReportDump*.txt
  "Fatal error!" in ShippingPC-RealmGameNoEditor.exe.
- NOT a launch-method problem. All three of these left the abort unchanged:
    * restoring the root EasyAntiCheat/Certificates/{base.bin,base.cer}
    * adding a RealmGame/CookedPC/ dir with the precompiled ShaderCodeStorage
    * launching via RealmEAC.exe (EAC bootstrapper) instead of ShippingPC direct
- Reproduces identically on BOTH Linux/Proton machines, both byte-perfect to
  the manifest => 6744 is broken as published. cluckers cannot reconstruct
  data the server never republished.
